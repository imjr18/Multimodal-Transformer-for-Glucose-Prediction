# Dataset Notes for the Non-Invasive Project

This project is designed to run end to end even when no external real-world dataset is present. If `noninvasive_glucose/data/raw/` is empty, the pipeline falls back to the synthetic cohort generated by `simulation/noninvasive_simulator.py`.

## Real-data slot: PhysioCGM

Place any licensed or approved PhysioCGM export files in `noninvasive_glucose/data/raw/`.

Expected high-level contents for a future adapter:

- synchronized biosignals or per-user recordings
- glucose reference labels used only for training
- timestamps aligned strongly enough to form 30-minute windows

This repository does not assume a single public PhysioCGM file schema. If your local copy uses CSV, Parquet, NPZ, or HDF5, keep the original files in `raw/` and add a dataset-specific ingestion adapter before replacing the synthetic pipeline.

## Why the code defaults to synthetic data

The non-invasive estimator needs multimodal inputs: HR, ECG-derived HRV features, EMG, EEG band powers, and CBF. Public datasets that contain all of these together with dense glucose labels are rare and often access-restricted. The synthetic cohort therefore serves two purposes:

- it lets the full training, calibration, uncertainty, and attribution pipeline run immediately
- it preserves control over physiological coupling so architecture changes can be studied without blocking on data access

## Recommended workflow

1. Start with the synthetic cohort and verify the pipeline end to end.
2. Add a real-data adapter that outputs the same processed window format used by `train_windows.pt`, `val_windows.pt`, and `test_windows.pt`.
3. Reuse the fine-tuning, calibration, and evaluation code unchanged once your adapter produces matching tensors and metadata.
